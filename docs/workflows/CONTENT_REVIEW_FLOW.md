# Content Review Flow

Este documento define o fluxo de criação e revisão de conteúdo.

Aplica-se a:

- carrosséis;
- posts;
- anúncios;
- mensagens;
- propostas;
- páginas;
- landing pages;
- textos institucionais;
- mensagens de WhatsApp;
- copies comerciais.

## Objetivo

Evitar erros de português, acentuação, espaçamento, clareza e repetição de erros pelos agentes.

## Regra principal

Conteúdo gerado deve passar por revisão antes de ser considerado aprovado.

## Fluxo recomendado

1. Gerar primeira versão.
2. Revisar ortografia.
3. Revisar acentos.
4. Revisar espaçamentos.
5. Revisar clareza.
6. Revisar tom.
7. Revisar formato.
8. Registrar correções.
9. Gerar versão corrigida.
10. Fazer nova revisão.
11. Aprovar ou repetir ciclo.

## Revisão ortográfica

Verificar:

- palavras erradas;
- acentos;
- concordância;
- pontuação;
- letras duplicadas;
- espaços extras;
- quebra de linha ruim.

## Revisão de formato

Verificar se o formato combina com o canal.

### WhatsApp

Seguir:

- `docs/brand/WHATSAPP_MESSAGE_STYLE.md`

### Carrossel

Verificar:

- pouco texto por slide;
- título forte;
- hierarquia clara;
- CTA;
- sequência lógica;
- ortografia;
- espaçamento.

### Proposta

Verificar:

- clareza;
- profissionalismo;
- preço;
- escopo;
- próximos passos.

## Registro de correções

Quando o agente encontrar erro recorrente, registrar em:

- `docs/project-memory/FINDINGS.md`

Exemplo:

- Agente costuma esquecer acento em “você”.
- Agente costuma deixar espaço duplo antes de pontuação.
- Agente costuma usar emoji mesmo quando padrão proíbe.
- Agente costuma usar `#` em mensagem de WhatsApp, mas o padrão proíbe.

## Aprovação

Status possíveis:

- não revisado;
- em revisão;
- aprovado;
- aprovado com ajustes;
- reprovado;
- precisa reescrever.

## Checklist de revisão

- [ ] Ortografia revisada?
- [ ] Acentos revisados?
- [ ] Espaçamentos revisados?
- [ ] Pontuação revisada?
- [ ] Tom adequado?
- [ ] Formato adequado ao canal?
- [ ] CTA claro?
- [ ] Sem emoji quando padrão proíbe?
- [ ] Sem Markdown errado para WhatsApp?
- [ ] Correções registradas em FINDINGS quando recorrentes?

## Regra para agentes

Nunca considerar conteúdo final aprovado sem uma etapa de revisão.

Se o usuário pedir texto rápido, ainda assim revisar antes de entregar.
