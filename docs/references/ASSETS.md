# Assets

Este documento organiza assets do projeto.

## Objetivo

Registrar onde estão logos, imagens, previews, ícones, PDFs, arquivos, referências e materiais importantes.

## Regra principal

Assets importantes devem ter origem, uso e status documentados.

## Tipos de asset

- logo;
- favicon;
- imagem Open Graph;
- imagem de preview;
- prints;
- PDFs;
- propostas;
- imagens de produto;
- ícones;
- referências visuais;
- arquivos de cliente;
- imagens geradas por IA;
- vídeos;
- documentos.

## Estrutura recomendada

| Nome | Tipo | Caminho/link | Uso | Status |
|---|---|---|---|---|
| A definir | A definir | A definir | A definir | A definir |

## Status possíveis

- rascunho;
- aprovado;
- substituir;
- pendente;
- antigo;
- final;
- não usar.

## Organização sugerida

Quando houver projeto web:

```txt
public/
  brand/
  images/
  og/
  icons/
  previews/
```

Quando houver storage:

```txt
storage/
  public/
  private/
  previews/
  uploads/
```

## Imagens de preview

Assets de preview devem seguir:

- `docs/brand/LINK_PREVIEW.md`

## Logo e favicon

Assets de logo/favicons devem seguir:

- `docs/brand/LOGO_AND_FAVICON.md`

## Arquivos privados

Arquivos privados devem seguir:

- `docs/security/STORAGE_SECURITY.md`
- `docs/storage/STORAGE_MANAGEMENT.md`
