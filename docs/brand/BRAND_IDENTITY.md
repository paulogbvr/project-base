# Brand Identity

Este documento define o padrão para identidade visual de projetos derivados da Project Base.

## Objetivo

Garantir que cada projeto tenha uma identidade visual clara, consistente e bem documentada antes de avançar demais na interface.

A identidade não precisa estar perfeita no início, mas precisa ter direção.

## Regra principal

Se o usuário não informar uma identidade visual, o agente deve partir do padrão base:

- monocromático premium;
- preto, branco e cinzas;
- dark mode e light mode;
- uma cor de acento a definir;
- layout limpo, forte e organizado.

Se a cor de acento não for informada, o agente deve perguntar.

## Informações principais

Preencher para cada projeto:

- Nome do projeto:
- Nome comercial:
- Descrição curta:
- Público:
- Tom:
- Cor principal:
- Cor de acento:
- Modo preferido inicial:
- Referências visuais:
- Restrições visuais:

## Nome do projeto

Separar:

### Nome técnico

Nome usado em pasta, repositório, variáveis e código.

Exemplo:

- `food-orders`
- `worklog`
- `project-base`

### Nome comercial

Nome exibido para cliente ou usuário final.

Exemplo:

- Central de Pedidos Rukka
- WorkLog
- Sistema Core

## Tom visual

Possíveis direções:

- premium;
- minimalista;
- técnico;
- editorial;
- institucional;
- operacional;
- moderno;
- monocromático;
- elegante;
- direto;
- sensorial.

## Padrão visual base

Quando não houver identidade definida, usar:

- fundo escuro elegante;
- modo claro preparado;
- contraste alto;
- cards bem definidos;
- bordas suaves;
- tipografia forte;
- detalhes discretos;
- sem excesso de cor;
- sem visual infantil;
- sem parecer template genérico.

## Cores

Registrar:

- cor principal;
- cor secundária;
- cor de acento;
- cor de sucesso;
- cor de alerta;
- cor de erro;
- cor neutra.

## Logo

A logo deve ser pensada junto com:

- sidebar;
- favicon;
- preview de link;
- metadata;
- tela de login;
- tela de loading;
- app shell.

Ver:

- `docs/brand/LOGO_AND_FAVICON.md`

## Preview de link

A identidade também precisa funcionar quando o link for compartilhado.

Ver:

- `docs/brand/LINK_PREVIEW.md`

## Mensagens

Se o projeto gerar mensagens para WhatsApp, seguir:

- `docs/brand/WHATSAPP_MESSAGE_STYLE.md`
