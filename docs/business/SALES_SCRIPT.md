# Sales Script

Este documento define roteiro de venda, apresentação ou demonstração.

## Objetivo

Ajudar a apresentar o projeto com clareza, sem começar pelo preço.

A ordem correta é:

1. Diagnóstico.
2. Problema.
3. Solução.
4. Demonstração.
5. Benefícios.
6. Proposta.
7. Próximo passo.

## Regra principal

Não vender apenas “site” ou “sistema”.

Vender transformação, organização, economia de tempo, profissionalismo, controle e resultado.

## Estrutura de apresentação

### 1. Abertura

Explicar que você preparou algo visual e prático.

Exemplo:

Eu não pensei nisso só como uma tela mais bonita. Pensei em uma central para organizar melhor a operação e melhorar a experiência de quem usa.

### 2. Diagnóstico

Mostrar o problema atual.

Exemplos:

- visual confuso;
- informações quebradas;
- fluxo ruim;
- falta de clareza;
- retrabalho;
- dependência de plataforma externa;
- pouca confiança para o cliente;
- dificuldade operacional.

### 3. Solução

Apresentar a solução como sistema próprio.

Exemplo:

A ideia é criar uma central própria para reunir acesso, informações, acompanhamento, atualizações e gestão em um só lugar.

### 4. Demonstração

Mostrar fluxo real.

Exemplo:

- abrir página;
- mostrar tela principal;
- executar ação;
- mostrar feedback;
- mostrar admin;
- mostrar link compartilhado;
- mostrar status;
- mostrar próxima fase.

### 5. Benefícios

Traduzir recurso em benefício.

Exemplo:

- cardápio bonito → melhora confiança;
- link compartilhado → cliente acompanha sozinho;
- status → menos dúvida;
- admin → mais controle;
- integração → menos retrabalho;
- pagamento → fluxo mais profissional.

### 6. Proposta

Apresentar planos.

Evitar começar com preço.

### 7. Próximo passo

Sempre fechar com uma ação clara:

- aprovar MVP;
- marcar próxima reunião;
- enviar dados;
- liberar acesso;
- escolher plano;
- iniciar implantação.
