# Current System Audit

Este documento registra análise do sistema atual, quando existir.

## Objetivo

Entender o estado atual antes de propor ou construir uma solução nova.

## Quando usar

Usar quando o cliente/projeto já tiver:

- site;
- sistema;
- cardápio;
- dashboard;
- planilha;
- app;
- fluxo manual;
- processo por WhatsApp;
- plataforma de terceiros;
- sistema antigo.

## Regra principal

Antes de construir o novo, entender o que está ruim no atual.

## Análise visual

Avaliar:

- primeira impressão;
- clareza;
- hierarquia;
- contraste;
- mobile;
- imagens;
- textos;
- loading;
- erros visíveis;
- aparência profissional.

## Análise de fluxo

Avaliar:

- caminho até ação principal;
- quantidade de cliques;
- clareza de botões;
- mensagens de erro;
- feedback;
- finalização;
- acompanhamento.

## Análise técnica superficial

Avaliar quando possível:

- URL;
- rotas;
- metadata;
- favicon;
- preview de link;
- responsividade;
- performance percebida;
- erros no front;
- textos quebrados;
- HTML aparecendo;
- imagens quebradas;
- formulários.

## Análise de segurança superficial

Avaliar:

- IDs previsíveis em URL;
- links públicos inseguros;
- dados sensíveis visíveis;
- páginas privadas acessíveis;
- metadata expondo dados;
- arquivos públicos indevidos.

## Pontos positivos

Registrar o que funciona bem.

## Pontos negativos

Registrar problemas.

## Oportunidades

Registrar melhorias possíveis.

## Prioridade

Classificar problemas:

- crítica;
- alta;
- média;
- baixa;
- observação.

## Saída esperada

Gerar resumo:

- principais problemas;
- melhorias rápidas;
- riscos;
- oportunidades;
- proposta de nova solução.
