# Carousel Skill

Skill auxiliar para criação de carrosséis.

Esta skill não faz parte do sistema principal do projeto. Ela é um recurso auxiliar para criar, revisar e aprovar carrosséis com mais segurança.

## Objetivo

Criar carrosséis com:

- copy forte;
- estrutura clara;
- sequência lógica;
- pouco texto por página;
- revisão ortográfica;
- revisão de acentos;
- revisão de espaçamentos;
- revisão de clareza;
- ciclo de aprovação.

## Quando usar

Usar para:

- Instagram;
- LinkedIn;
- apresentação curta;
- conteúdo educativo;
- conteúdo institucional;
- campanha;
- lançamento;
- explicação de produto;
- storytelling de marca.

## Fluxo recomendado

1. Definir objetivo do carrossel.
2. Definir público.
3. Definir tom.
4. Criar estrutura página por página.
5. Escrever primeira versão.
6. Revisar ortografia.
7. Revisar acentos.
8. Revisar espaçamento.
9. Revisar clareza.
10. Revisar hierarquia.
11. Gerar versão final.
12. Registrar correções recorrentes.

## Estrutura sugerida

### Página 1

Hook forte.

Objetivo:

- parar a rolagem;
- gerar curiosidade;
- deixar claro o tema.

### Páginas intermediárias

Desenvolver a ideia.

Cada página deve ter:

- um ponto principal;
- texto curto;
- sequência lógica;
- visual fácil de entender.

### Última página

Fechamento e CTA.

Exemplos:

- chamar para WhatsApp;
- pedir comentário;
- convidar para conhecer;
- direcionar para link;
- reforçar autoridade;
- gerar ação.

## Revisão obrigatória

Antes de aprovar, revisar:

- ortografia;
- acentuação;
- concordância;
- pontuação;
- espaços extras;
- quebras de linha;
- repetição;
- clareza;
- CTA;
- consistência visual.

## Registro de erros

Se houver erro recorrente, registrar em:

- `docs/project-memory/FINDINGS.md`

Exemplos:

- agente esqueceu acento;
- agente colocou espaço duplo;
- agente usou emoji quando padrão proíbe;
- agente exagerou no texto por página;
- agente perdeu o tom da marca.

## Padrão para aprovação

Status possíveis:

- rascunho;
- em revisão;
- aprovado com ajustes;
- aprovado;
- reprovado;
- reescrever.

## Prompt base

Crie um carrossel seguindo este fluxo:

1. Pergunte o objetivo, público, tema, tom e CTA.
2. Monte a estrutura página por página.
3. Escreva a primeira versão.
4. Revise ortografia, acentos, espaçamento e clareza.
5. Aponte correções feitas.
6. Gere a versão final.
7. Registre aprendizados recorrentes em `docs/project-memory/FINDINGS.md`, se houver.
